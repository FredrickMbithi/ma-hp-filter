## Conclusions

Dated 2026-03-11.

**Negative findings:**

- Multi-factor is **worse IS** than baseline (Sharpe 0.24 vs 0.54) — the extra features add noise, not signal
- Both strategies **fail catastrophically OOS** (Sharpe ≈ −1.2)
- The high trade count (504 IS vs 104 baseline) is 5× more cost drag with no return benefit
- Feature ICs at H=24 are essentially zero (< 0.012); combined signal p-value > 0.77 (noise)

**Prior research discrepancy:**
Prior experiment logs reported IS Sharpe ≈ 3.7. This run with the causal HP filter gives IS Sharpe = 0.54. Most likely explanation: prior experiments used the **two-sided `apply_hp_filter`** (look-ahead bias — future prices used when computing the HP trend). The causal rolling-window HP filter used here (past data only) produces the correct, bias-free result. All prior IS Sharpe numbers computed with the two-sided filter should be considered invalid.

**Do not proceed to live testing with current signals.**
